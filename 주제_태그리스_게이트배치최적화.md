# 태그리스 전용 게이트(Express Lane) 지정의 통행비용 절감 효과 분석 — 성수역 보행 시뮬레이션

## 1. 연구 배경

- 서울교통공사, 1~8호선 태그리스 시스템 **단계적 도입** 추진 중 (시범사업 진행)
- 도입 과정: 역당 일부 게이트에 태그리스 하드웨어를 설치하고, 점진적으로 설치 게이트 수를 확대
- 태그리스 하드웨어가 설치된 게이트는 현재 **태그/태그리스 겸용**으로 운영 (역당 1~2대)
- 나머지 게이트는 태그리스 미설치 → **태그 전용**
- 태그리스 자체의 통행비용 절감 효과는 이미 검증됨

## 2. 문제 제기

- 태그리스 하드웨어가 설치된 게이트(K대)를 **겸용으로 운영**하면, 서비스 시간이 상이한 두 이용자 유형이 혼재:
  - 태그 이용자: 정지 → 카드 탭 → 문 열림 → 통과 (약 2~3초)
  - 태그리스 이용자: 감속 없이 연속 통과 (약 1초 이하)
- 겸용 게이트에서 태그리스 이용자는 선행 태그 이용자의 정지·탭 동작으로 인해 **대기행렬에 갇힘**
- 태그리스의 무정차 통과 이점이 상실 → 기대 서비스 수준과 실제 서비스 수준의 괴리 발생
- **고속도로 하이패스 전용 차로**와 동일한 구조:
  - 하이패스 단말기가 설치된 차로를 겸용(일반+하이패스)으로 운영 → 일반 차량이 정차하면 하이패스 차량도 대기
  - **전용 차로로 분리** → 하이패스 차량 무정차 통과 → 현재 전국 고속도로에서 채택된 운영 방식
- 지하철 게이트에서도 동일한 **전용 분리 운영의 정량적 효과**에 대한 근거 필요

## 3. 연구 목적

태그리스 단계적 도입 과정에서, 태그리스 하드웨어가 설치된 게이트를 **겸용으로 운영할 것인가, 태그리스 전용(express lane)으로 운영할 것인가**를 정량적으로 비교하고, **태그리스 이용자 비율 변화에 따른 최적 전용 게이트 수(K\*)를 도출**하여 단계적 운영 정책 근거를 제시한다.

## 4. 연구 방법

### 4-1. 시뮬레이션 모델

- **도구**: JuPedSim (오픈소스 보행 시뮬레이터, Python/C++)
- **모델**: Collision-Free Speed Model V2 (CFSM V2, Tordeux et al. 2016; Xu et al. 2019)
- **게이트 선택**: 다항 로짓 모형 (MNL, Gao et al. 2019 LRP 기반)
- **대상역**: 성수역 2호선 서쪽 대합실
  - 환승 없음, 역사 구조 단순
  - 승객/게이트 비율 상위 (일평균 105,574명 / 39개 게이트 = 2,707명/게이트)
- **공간 범위**: 계단 → 대합실 → 게이트

### 4-2. 성수역 게이트 현황

| 구분 | 수량 |
|------|------|
| 턴스타일 개집표기 | 31대 |
| 우대자용 개집표기 | 8대 |
| **합계** | **39대** |

- 서쪽 대합실 모델: 7개 게이트 (시뮬레이션 대상)

### 4-3. 이용자 유형 및 게이트 서비스 시간

| 이용자 유형 | 통과 방식 | 서비스 시간 |
|------------|----------|-----------|
| 태그 이용자 | 정지 → 카드 탭 → 문 열림 → 통과 | LogNormal(μ_ln=0.568, σ_ln=0.5), [0.8, 3.7]s |
| 태그리스 이용자 | 감속 없이 연속 통과 | 0s (민감도: 0/0.5/1.0s) |

### 4-4. 시나리오 설계

#### 기본 시나리오 (현행 겸용 운영)
- 태그리스 하드웨어 설치 게이트 K대를 **겸용(태그+태그리스)**으로 운영
- 나머지 게이트는 태그 전용 (하드웨어 미설치)
- 겸용 게이트에서 태그/태그리스 이용자 혼재 → 태그리스 이용자 대기 발생

#### 개선 시나리오 (express lane 전용 운영)
- 동일한 K대 게이트를 **태그리스 전용(express lane)**으로 운영
- 태그리스 전용 게이트: 태그리스 이용자만 선택 가능 → 무정차 연속 통과
- 나머지 게이트: 태그 전용 (현행 유지)
- **추가 하드웨어 설치 없이 운영 정책(안내판/소프트웨어)만 변경**하는 현실적 방안

#### 실험 요인

| 요인 | 수준 | 비고 |
|------|------|------|
| 게이트 운영 방식 | 전체 겸용 / express lane 분리 | 2수준 |
| 태그리스 이용자 비율(p) | 10%, 20%, 40%, 60%, 80% | 5수준 |
| 전용 게이트 수(K) | 1, 2, 3대 | 3수준 |
| 시간대 | 첨두 / 비첨두 | 2수준 |
| **합계** | | **2×5×3×2 = 60개 시나리오** |

### 4-5. 게이트 선택 의사결정 모델

보행자의 게이트 선택은 **다항 로짓 모형(MNL)**으로 모사:

- Gao et al. (2019) LRP(Least Resistance Path) 기반 효용함수
- 효용함수: `U_gate = β₁ × 거리 + β₂ × 대기열 길이 + β₃ × 서비스 시간`
- 거리 추정: 순서보존 노이즈 (Gao eq.4+5)
- 대기열 인지: 인지 오차 모형 (Gao eq.7)
- 보행자 성격 유형: adventurous / conserved / mild (1:1:1)
- 3단계 의사결정: 3.0m → 1.7m → 1.0m

#### 이용자 유형별 선택 집합(Choice Set) 차별화 — 연구 핵심

| 시나리오 | 태그 이용자 Choice Set | 태그리스 이용자 Choice Set |
|----------|----------------------|--------------------------|
| 기본 (겸용 운영) | 태그전용 게이트 + 겸용 게이트 | 겸용 게이트만 |
| 개선 (express lane) | 태그 전용 게이트만 | **태그리스 전용 게이트만** |

- 기본 시나리오: 태그리스 이용자는 겸용 게이트에서 태그 이용자와 혼재 → **대기 발생**
- 개선 시나리오: 이용자 유형별 게이트 완전 분리 → **혼재 제거**
- 겸용 게이트의 태그리스 이용자 비율을 외생적으로 가정하지 않고, **MNL 모형이 시뮬레이션 결과로 산출**

### 4-6. 입력 데이터

| 데이터 | 출처 | 비고 |
|--------|------|------|
| 성수역 시간대별 승하차인원 | 서울교통공사 공공데이터 | 첨두/비첨두 구분 |

| 역사 평면도 | 서울교통공사 도면 또는 현장 측정 | JuPedSim 공간 구현용 |
| 계단/에스컬레이터 용량 | TCQSM 문헌값 | |

### 4-7. 평가 지표

| 지표 | 정의 | 목적 |
|------|------|------|
| 평균 대기시간 | 게이트 큐 진입 ~ 통과 완료 | 이용자 체감 비용 |
| 최대 대기행렬 길이 | 게이트 앞 최대 큐 길이 | 물리적 공간 점유 |
| 총 통과시간 | 계단 진입 ~ 게이트 통과 | 전체 동선 비용 |
| 게이트 균형도 | 게이트별 이용률 편차 | 특정 게이트 쏠림 여부 |
| 보행자 밀도/LOS | 게이트-계단 구간 인/m² | 서비스 수준 평가 |
| 처리량 | 단위시간당 통과 인원 | 시스템 용량 |

## 5. 기대 결과

1. **겸용 운영 vs express lane 전용 운영의 통행비용 비교**
   - 태그리스 비율별 대기시간, 통과시간 차이 정량화
   - 겸용 게이트 내 이질적 서비스율 혼합의 비효율성 실증

2. **비율별 최적 전용 게이트 수(K\*) 곡선 도출**
   - "태그리스 비율 p%일 때 K*대를 express lane으로 지정하라"
   - 단계적 운영 전환 로드맵 제시

3. **전용 게이트 전환의 임계점 도출**
   - "태그리스 비율 X% 이상에서 express lane 지정이 유의미한 효과"

4. **정책 제언**
   - 추가 하드웨어 설치 없이 운영 정책만으로 달성 가능한 통행비용 절감 효과 제시
   - 서울교통공사 태그리스 단계적 확대 시 게이트 운영 방식 결정 근거 자료

## 6. 추후 연구

### 6-1. 실시간 통과패턴 기반 동적 게이트 전환 (Adaptive Gate Designation)

- 전체 역 태그리스 비율은 백엔드에서 파악 가능하지만, **특정 게이트 앞 실시간 혼합 비율**은 별도 추정 필요
- 게이트별 통과시간 패턴(서비스 시간 분포)에서 태그리스 비율을 실시간 추정
- 추정된 비율이 임계값을 초과하면 해당 게이트를 **자동으로 express lane으로 전환**
- 고속도로 가변차로 운영과 동일한 개념: 시간대/혼잡도에 따라 전용 게이트 수를 유동적으로 조정
- 본 연구의 정적 최적화 결과를 동적 제어 규칙으로 확장

### 6-2. 게이트 처리량 증가에 따른 하류 병목 전이 분석 (Bottleneck Shift)

- Express lane 도입으로 게이트 통과 속도가 향상되면, **병목이 게이트에서 하류(계단/에스컬레이터)로 전이**될 가능성
- 본 연구 시뮬레이션 공간이 게이트~계단을 포함하므로, 계단 앞 밀도(인/m²) 변화로 병목 전이 여부 확인 가능
- 게이트 최적화의 **시스템 전체 효과** 검증: 부분 최적화가 전체 최적화인지 확인
- 평가지표의 "보행자 밀도/LOS"를 게이트 구간과 계단 구간으로 분리하여 비교

### 연구 차별성
- 이질적 서비스율(태그/태그리스) 이용자 **혼합류에서 express lane 분리**의 통행비용 비교 → 선행연구 없음
- 게이트 **서비스율 차이에 기반한 운영 정책 최적화** → 연구 없음
- **하드웨어 변경 없이 운영 정책만으로** 달성 가능한 효과 정량화 → 연구 없음
- MNL 기반 이용자 유형별 **Choice Set 차별화** → 게이트 선택 연구에서 시도 없음
